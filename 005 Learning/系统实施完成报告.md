## 🎯 系统特性详解

### 1. 技术笔记系统（Tech Note）

**Frontmatter 字段：**
```yaml
tags: [tech, bigdata]
topic:              # 主题（Spark、Flink、SQL 等）
difficulty:         # 初级/中级/高级
status:             # 学习中/复习中/已掌握
progress: 0         # 掌握度 0-100%
source:             # 学习来源
related: []         # 关联笔记
```

**内容结构：**
- 📊 掌握度进度条（meta-bind progressBar）
- 🎯 核心概念（一句话定义）
- 📖 原理详解（基本原理、工作流程、关键特性）
- 💻 代码示例
- 🔧 应用场景
- ⚠️ 常见问题 & 坑
- 🔗 关联笔记

**使用场景：**
- 学习新的大数据技术
- 整理技术知识点
- 建立知识体系

---

### 2. 代码片段系统（Snippet）

**Frontmatter 字段：**
```yaml
tags: [snippet]
language:           # Python/SQL/Shell/Scala/Java
topic:              # 所属主题
description:        # 一句话描述
```

**内容结构：**
- 📝 用途说明
- 💻 代码块（带语言标注）
- 🔧 参数说明（表格）
- 📤 示例输出
- ⚠️ 注意事项

**使用场景：**
- 保存可复用的代码片段
- 快速查找常用命令
- 建立个人代码库

---

### 3. 学习日志系统（Learning Log）

**Frontmatter 字段：**
```yaml
tags: [learning-log]
topic:              # 今日学习主题
duration: 0         # 学习时长（分钟）
difficulty:         # 简单/中等/困难
mood: 3             # 学习状态 1-5
summary:            # 一句话总结
```

**内容结构：**
- 📊 学习状态（时长、难度、心情）
- 📚 今日学了什么（bullet points）
- 💻 代码/命令记录
- 🐛 遇到的问题 & 解决方案
- 📋 明日计划
- 🔗 关联笔记

**使用场景：**
- 每日学习记录
- 追踪学习进度
- 反思学习效果

---

### 4. 技术案例系统（Tech Case）

**Frontmatter 字段：**
```yaml
tags: [case, bigdata]
topic:              # 技术主题
tech-stack: []      # 技术栈列表
difficulty:         # 初级/中级/高级
progress: 0         # 完成度 0-100%
start-date:         # 开始日期
deadline:           # 截止日期
```

**内容结构：**
- 🎯 案例进度（完成度、时间设置）
- 📖 业务背景（问题描述、业务需求）
- 🏗️ 技术方案选型（方案对比、最终选择）
- 🔨 实现过程（步骤 checklist）
- 💻 核心代码
- ⚠️ 遇到的坑
- 🎓 结果 & 收获
- 🔗 关联资源

**使用场景：**
- 记录实战项目
- 总结最佳实践
- 积累架构经验

## 📊 Learning Dashboard 功能详解

### 实时数据统计

```dataviewjs
// 自动统计以下指标：
- 总笔记数
- 已掌握笔记数
- 学习中笔记数
- 复习中笔记数
- 平均掌握度
```

### 按主题分组

自动从所有技术笔记中提取 `topic` 字段，按主题分组显示：
- 每个主题的笔记数
- 平均掌握度进度条
- 状态分布（学习中/复习中/已掌握）

### 按难度分类

技术笔记自动按难度分类显示：
- 🟢 初级（beginner）
- 🟡 中级（intermediate）
- 🔴 高级（advanced）

### 按语言分类

代码片段自动按编程语言分类：
- Python
- SQL
- Shell
- Scala
- Java
- 其他

### 学习热力图

基于学习日志的 `duration` 字段，生成热力图显示：
- 每日学习时长
- 学习强度可视化
- 学习习惯分析

## 🚀 快速开始指南

### 创建技术笔记

1. 打开 Learning Dashboard
2. 点击"新建技术笔记"按钮
3. 选择 `Tech Note Template`
4. 填写 Frontmatter：
   - `topic`: 技术名称（如 "Spark"）
   - `difficulty`: 难度等级
   - `status`: 学习状态
5. 完成后自动移动到 `005 Learning/BigData/`

### 创建代码片段

1. 打开 Learning Dashboard
2. 点击"新建代码片段"按钮
3. 选择 `Snippet Template`
4. 填写 Frontmatter：
   - `language`: 编程语言
   - `topic`: 所属主题
   - `description`: 片段用途
5. 完成后自动移动到 `005 Learning/Snippets/`

### 记录学习日志

1. 打开 Learning Dashboard
2. 点击"新建学习日志"按钮
3. 选择 `Learning Log Template`
4. 填写 Frontmatter：
   - `topic`: 今日学习主题
   - `duration`: 学习时长
   - `mood`: 学习状态
5. 完成后自动移动到 `002 Journal/`

### 记录技术案例

1. 打开 Learning Dashboard
2. 点击"新建案例笔记"按钮
3. 选择 `Tech Case Template`
4. 填写 Frontmatter：
   - `topic`: 技术主题
   - `tech-stack`: 技术栈
   - `difficulty`: 难度等级
5. 完成后自动移动到 `005 Learning/Cases/`

## 📈 学习追踪方式

### 方式 1：掌握度进度条

每个技术笔记都有 `progress` 字段（0-100%），通过 meta-bind progressBar 可视化显示掌握度。

### 方式 2：学习状态标签

每个技术笔记都有 `status` 字段：
- `learning` — 学习中
- `reviewing` — 复习中
- `mastered` — 已掌握

### 方式 3：学习日志记录

每日记录学习时长、难度、心情，形成学习热力图和时间线。

### 方式 4：Dashboard 统计

Learning Dashboard 自动统计：
- 总笔记数和掌握度分布
- 按主题的进度统计
- 按难度的分类统计
- 本周学习热力图

## 💡 最佳实践建议

### 1. 笔记命名规范

**技术笔记：** `[技术名] - [核心概念]`
- 示例：`Spark - RDD 核心原理`、`Flink - 时间窗口机制`

**代码片段：** `[语言] - [用途]`
- 示例：`Python - Pandas 数据清洗`、`SQL - Hive 分区优化`

**学习日志：** `学习日志 - [日期] - [主题]`
- 示例：`学习日志 - 2026-05-16 - Spark SQL`

**技术案例：** `[项目名] - [技术主题]`
- 示例：`用户行为分析 - Spark + Flink`

### 2. 关联笔记的使用

在 `related` 字段中添加相关笔记的 wikilink：
```yaml
related:
  - "[[Spark - RDD 核心原理]]"
  - "[[Spark - DataFrame API]]"
```

### 3. 定期复习

- 每周复习一次 `status: learning` 的笔记
- 将掌握的笔记标记为 `status: mastered`
- 追踪 `progress` 字段的增长

### 4. 学习日志的坚持

- 每日记录学习内容和时长
- 在学习热力图中看到自己的学习习惯
- 定期回顾学习日志，反思学习效果

### 5. 代码片段的积累

- 遇到有用的代码就保存为片段
- 定期整理和优化片段
- 在新项目中快速复用
